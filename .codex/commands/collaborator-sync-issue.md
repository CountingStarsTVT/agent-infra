---
description: Sync task progress to the corresponding GitHub Issue as a comment
argument-hint: <task-id>
---

Sync the progress of task $1 to the corresponding GitHub Issue as a comment.

Execute the following steps:

1. Find the task file:
   Search in order:
   - .ai-workspace/active/$1/task.md (priority)
   - .ai-workspace/blocked/$1/task.md
   - .ai-workspace/completed/$1/task.md
   If not found, inform the user the task does not exist.

2. Read task info:
   Get issue_number from task.md (if missing, ask user to specify), title, current_step, status.

3. Read context files (if they exist):
   - analysis.md, plan.md, implementation.md, review.md
   Extract key progress and decisions.

4. Generate progress summary in Markdown format:
   ```markdown
   ## Task Progress Update

   **Task ID**: $1
   **Updated**: <current time>
   **Status**: <status description>

   ### Completed
   - [x] <completed steps and key points>

   ### Current Progress
   <current step details>

   ### Next Steps
   <next step plan>

   ---
   *Auto-generated by Codex*
   ```

5. Sync to Issue:
   ```bash
   gh issue comment <issue-number> --body "<summary>"
   ```

6. Inform user:
   - Display synced Issue link
   - Output summary of synced content

**Note**: Keep summaries concise and human-readable. Avoid syncing too frequently - sync after completing a full stage.
